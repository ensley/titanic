Titanic: Machine Learning from Disaster¶

This is a use case for ccdspy, a data science project template for Python.

It implements a solution to the Titanic Kaggle competition that ranks in the top 15-20% on the leaderboard.

Installation¶

Run

pip install -r requirements.txt

Usage¶

The easiest way to reproduce the project is by using the make directives.

Available rules:

clean               Delete all models and data
data                Transform raw data
docs                Build documentation and copy it to the docs folder
githooks            Set up githooks
models              Train models
mostlyclean         Delete all models and data, except for the raw downloads
predictions         Create predictions
raw_data            Pull raw data from Kaggle

The project pipeline is:

  1. Download raw data from Kaggle.

  2. Clean the raw data, create new features and transform existing ones.

  3. Train the models using the training data.

  4. Generate predictions using the test data.

Those four steps correspond to the following directives:

  1. make raw_data

  2. make data

  3. make models

  4. make predictions

For each of these steps, make will run as many of the preceding steps as necessary to ensure the required input files exist. This means that, starting from a completely empty (or nonexistent) data/ directory, running make predictions will perform the entire pipeline from raw data download to generating predictions.

See Makefile rules for details on each of these directives.
